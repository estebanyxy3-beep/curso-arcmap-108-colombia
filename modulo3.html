🎯 Objetivo del módulo

Entender dónde están las capas y cómo hacer que coincidan

El estudiante aprenderá a revisar el sistema de coordenadas de una capa, diferenciar coordenadas geográficas y proyectadas, y preparar las capas del municipio para trabajar en un mismo sistema.

Idea clave

Un sistema de coordenadas es como la dirección de una casa, pero aplicada al territorio. Sin esa dirección, ArcMap no sabe dónde ubicar correctamente una capa.

📌

Producto final del módulo

Capas vectoriales revisadas, corregidas y proyectadas al sistema adecuado para análisis espacial en Colombia, guardadas en la carpeta de resultados.

  • Spatial Reference revisado.
  • Capas sin proyección identificadas.
  • Define Projection usado solo cuando corresponde.
  • Project usado para transformar coordenadas.

❓ ¿Por qué importa?

Si las coordenadas están mal, el análisis sale mal

Cuando dos capas no tienen el mismo sistema de referencia, pueden aparecer lejos, superpuestas incorrectamente o medir distancias equivocadas.

Ejemplo fácil

Si una capa está en grados y otra en metros, ArcMap puede mostrarlas desalineadas. También puedes calcular un buffer de “100” pensando que son metros, pero en realidad la capa puede estar en grados.

Ilustración didáctica: cuando las capas comparten sistema de coordenadas, se alinean correctamente para análisis.

🧭 Tipos de coordenadas

Coordenadas geográficas vs. coordenadas proyectadas

Esta es la explicación central del módulo. El estudiante debe comprender que no todas las coordenadas se expresan igual.

🌐 Coordenadas geográficas

Ubican puntos usando latitud y longitud sobre la Tierra. Sus unidades son grados decimales.

Ejemplo: Latitud 7.8790° N
Longitud -72.5078° W
  • Usan grados.
  • Son comunes en GPS.
  • WGS84 es el ejemplo más conocido.
  • No son ideales para medir áreas o distancias.

📏 Coordenadas proyectadas

Ubican puntos sobre un plano. Sus unidades normalmente son metros, por eso son mejores para medir.

Ejemplo: Este 4.850.000 m
Norte 2.100.000 m
  • Usan metros.
  • Permiten medir distancias y áreas.
  • Son necesarias para buffers y geoprocesamiento.
  • En Colombia se recomienda MAGNA-SIRGAS / Origen-Nacional para cartografía oficial.

Las coordenadas geográficas se expresan en grados; las proyectadas se expresan en metros y son mejores para análisis espacial.

🌎 Datum

El datum es la referencia de la Tierra que usa el mapa

El datum define el modelo matemático de la Tierra y el punto de referencia desde el cual se calculan las coordenadas. Dos capas pueden tener valores parecidos, pero si usan datum diferentes, no siempre coinciden exactamente.

Cómo explicarlo sencillo

Imagina que varias personas dibujan la misma montaña usando reglas distintas. Todas dibujan la montaña, pero las medidas pueden variar. El datum es esa regla de referencia.

Ilustración: dos referencias distintas pueden ubicar el mismo punto con pequeñas diferencias.

🗺️ Proyección cartográfica

Proyectar es pasar la Tierra curva a un plano

Toda proyección transforma la superficie curva de la Tierra en un plano. Esto permite trabajar en mapas impresos o digitales, pero siempre introduce algún tipo de deformación.

📐

Deforma algo

Puede deformar áreas, formas, distancias o direcciones. No existe una proyección perfecta para todo.

📏

Permite medir

Cuando una capa está proyectada en metros, puedes calcular distancias, áreas, buffers y análisis de proximidad.

🇨🇴

Debe ser adecuada al país

Para Colombia se usa MAGNA-SIRGAS / Origen-Nacional en cartografía oficial moderna y muchos cursos lo reconocen como CTM12.

La proyección permite pasar de una superficie curva a una grilla plana para hacer mediciones.

🇨🇴 Sistemas para Colombia

WGS84, MAGNA-SIRGAS y Origen-Nacional

En Colombia encontrarás diferentes sistemas según el origen del dato. Lo importante es reconocerlos, no mezclarlos sin revisar y proyectar cuando sea necesario.

Sistema Tipo Unidades Uso común Precaución
WGS84 Coordenadas geográficas Grados decimales GPS, imágenes satelitales, datos globales, plataformas web. No es ideal para medir áreas, distancias o buffers.
MAGNA-SIRGAS Datum / sistema de referencia geodésico Depende de la proyección usada Referencia nacional moderna para datos oficiales de Colombia. Verifica si la capa está geográfica o proyectada.
MAGNA-SIRGAS / Origen-Nacional Coordenadas proyectadas Metros Cartografía oficial moderna, catastro, análisis espacial nacional. En muchos materiales se conoce como CTM12; su código EPSG moderno es 9377.
Orígenes Gauss-Krüger antiguos o locales Coordenadas proyectadas Metros Cartografía histórica o local. No asumir que es igual a Origen-Nacional; revisar siempre.

Resumen importante

Recuerda: WGS84 sirve para ubicar globalmente; MAGNA-SIRGAS es la referencia nacional; y MAGNA-SIRGAS / Origen-Nacional permite trabajar en metros para análisis espacial en Colombia.

🔁 Herramientas críticas

Define Projection no es lo mismo que Project

Esta diferencia debe quedar totalmente clara. Es uno de los errores más comunes en ArcMap.

📌 Define Projection

Se usa cuando la capa ya tiene coordenadas correctas, pero ArcMap no sabe qué sistema tienen porque falta la definición o el archivo .prj.

  • No transforma coordenadas.
  • No mueve la capa realmente.
  • Solo escribe o asigna la etiqueta del sistema.
  • Se usa cuando conoces con seguridad el sistema original.

🔁 Project

Se usa cuando quieres cambiar una capa de un sistema de coordenadas a otro, por ejemplo de WGS84 a MAGNA-SIRGAS / Origen-Nacional.

  • Sí transforma coordenadas.
  • Crea una nueva capa de salida.
  • Debe usarse para unificar capas antes del análisis.
  • Es la herramienta correcta para pasar de grados a metros.

Define Projection etiqueta; Project transforma. No se deben intercambiar.

🖥️ En ArcMap

Cómo revisar el sistema de coordenadas

Antes de cualquier análisis, revisa el sistema de cada capa y del Data Frame.

1️⃣

Clic derecho sobre la capa

En Table of Contents, haz clic derecho sobre la capa que quieres revisar.

2️⃣

Properties

Abre la ventana de propiedades de la capa.

3️⃣

Source

En la pestaña Source, busca el campo Spatial Reference.

4️⃣

Revisar unidades

Si ves grados, es geográfica. Si ves metros, está proyectada.

5️⃣

Data Frame

También revisa Data Frame Properties → Coordinate System.

6️⃣

Unificar

Proyecta las capas que lo requieran y guarda nuevas salidas en Resultados.

⚠️ Errores comunes

Problemas frecuentes con coordenadas

⚠️
Confundir Define Projection con Project.

Define Projection no transforma coordenadas; Project sí.

⚠️
Hacer buffer en grados.

Antes de buffers o distancias, proyecta la capa a metros.

⚠️
Trabajar con capas en sistemas diferentes.

Unifica el sistema antes de geoprocesar.

⚠️
Asignar un sistema al azar.

No uses Define Projection sin conocer el sistema original.

⚠️
No guardar la capa proyectada.

Project crea una nueva capa; guárdala con nombre claro.

⚠️
No revisar el Data Frame.

El Data Frame también tiene sistema de coordenadas y afecta la visualización.

⚠️
Confiarse porque “se ve bien”.

Que se vea alineado no significa que esté listo para medir. Revisa unidades y sistema.
